EDR / XDR / MDR 入門學習地圖

給非資安背景也看得懂的實務導向版本：先理解，再選型，最後能落地

你會學到： 名詞完整名稱、AV/防火牆/EDR 差異、現代攻擊流程、品牌比較、90 天導入建議。
適合對象： IT、MIS、主管、採購、剛接觸資安的人。

A. 一頁看懂：先建立共同語言

先把縮寫背後的完整名稱釐清，後面才不會混淆。

AV

Anti-Virus（傳統防毒軟體）

以病毒特徵碼（Signature）為主，擅長已知惡意檔案，但對全新未知威脅反應較慢。

EDR

Endpoint Detection and Response（端點偵測與回應）

部署在端點主機上，持續蒐集行為紀錄，偵測異常並可執行隔離、終止程序等回應動作。

XDR

Extended Detection and Response（延伸偵測與回應）

把端點、網路、雲端、郵件、身分等資料關聯分析，讓事件調查更完整。

MDR

Managed Detection and Response（託管偵測與回應）

由外部專家團隊 24/7 代管監控與應變，適合資安人力不足的組織。

MITRE ATT&CK 精簡對照表（入門）

階段 / Technique 常見手法 可偵測訊號 建議控制（EDR/SIEM/MFA/網段隔離）
Initial Access
T1566 Phishing		 釣魚郵件誘導點擊連結或開啟惡意附件。 郵件來源異常、Office 啟動腳本、連到新註冊網域。 EDR 偵測 Office 子程序異常；SIEM 關聯郵件與端點事件；MFA 防帳號被盜用。
Execution
T1059 Command and Scripting Interpreter		 PowerShell / cmd / bash 執行惡意腳本。 編碼指令、可疑參數（如 -enc）、父子程序鏈異常。 EDR 阻擋高風險腳本；SIEM 建立指令基線與告警；限制高權限終端可執行腳本。
Persistence
T1547		 透過啟動項、登錄機碼或服務維持存活。 Run/RunOnce 新增、排程工作異常、未知服務常駐。 EDR 監控持久化行為；SIEM 追蹤設定變更；網段隔離關鍵伺服器降低橫向擴散。
Credential Access
T1003		 Dump LSASS / SAM 取得帳密雜湊。 對 LSASS 記憶體存取、Mimikatz 特徵、異常提權行為。 EDR 阻擋憑證傾印；SIEM 監看高風險提權；MFA 降低帳密外洩後的成功登入率。
Lateral Movement
T1021		 使用 RDP/SMB/WinRM 在內網橫向移動。 非慣常管理連線、夜間跨網段登入、同帳號多主機快速登入。 SIEM 偵測東西向流量異常；EDR 監控遠端工具濫用；網段隔離限制管理協定可達範圍。
Exfiltration
T1041		 透過 C2 通道或雲端服務外傳資料。 異常大量上傳、罕見目的地連線、壓縮加密檔案後對外傳輸。 EDR/DLP 偵測敏感檔案外傳；SIEM 關聯流量與檔案存取；網段隔離與出口管制降低外傳成功率。
現在資安環境為什麼改變？ 遠端辦公增加端點暴露、勒索攻擊工業化（RaaS）、無檔案攻擊上升、資料外洩與法規壓力並存。

B. 差異比較：AV、防火牆、EDR 各自做什麼？

AV vs EDR

核心差異

  • AV：看「檔案像不像已知病毒」。
  • EDR：看「行為是不是攻擊模式」。
  • AV：多偏向預防；EDR：含偵測、調查、回應。
入門重點：AV 不是沒用，而是單獨使用已不足以應對新型攻擊。

防火牆（Firewall） vs EDR

  • 防火牆：守網路邊界，管流量進出（外到內）。
  • EDR：守每台主機內部，監看程序、命令、權限變化。
最佳實務：防火牆 + EDR + 多因子驗證 + 備份還原 + 事件演練。

快速選型建議（入門版）

情況優先選擇原因
剛起步，先想保護端點不被勒索EDR最快建立端點可視性與隔離能力。
工具很多但告警很亂、跨系統難追查XDR把多來源訊號關聯，提升調查效率。
人力不足、無法 24/7 值班MDR + EDR/XDR借助外部 SOC 團隊補齊監控與應變。

C. EDR 如何抓未知威脅？（最重要）

持續監控：代理程式蒐集 Process、File、Registry、Network、Script、登入事件。
行為分析：用規則 + 情資 + 模型比對異常行為鏈，而不只看檔案簽名。
回應處置：終止惡意程序、隔離主機、封鎖 IOC、通知管理者進行調查。

情境模擬：全新惡意 Word 檔案（Zero-day）

傳統 AV 可能結果：
  • 沒有特徵碼時可能先放行。
  • 需等待雲端樣本分析與病毒庫更新。
EDR 可能結果：
  • 發現 Word 啟動異常腳本與大量加密行為。
  • 立即阻斷程序並隔離端點，降低擴散風險。

初學者常見誤解

有 EDR 就不用防火牆了嗎？

不是。兩者防護層次不同，邊界與端點應同時部署。

MDR 是產品還是服務？

主要是服務，由外部團隊代管偵測與回應流程。

XDR 一定比 EDR 好嗎？

不一定。若你連端點治理都尚未成熟，先把 EDR 打好更務實。

D. 應該知道的專業術語（實務常用）

這份術語表可作為跨部門溝通底稿，建議 IT、資安、主管使用同一套定義。

術語全名 / 中文你要知道的重點
IOCIndicator of Compromise（入侵指標）惡意 IP、網域、檔案雜湊等可觀察痕跡，可用於封鎖與追查。
TTPTactics, Techniques, and Procedures（戰術技術程序）攻擊者行為模式；比單一 IOC 更耐用，適合長期偵測策略。
C2Command and Control（命令與控制）受害主機與攻擊者伺服器通訊通道，常見於資料外傳與遠端操控。
LOLBinsLiving off the Land Binaries（系統原生工具濫用）攻擊者濫用 PowerShell、certutil 等合法工具以躲避偵測。
SIEMSecurity Information and Event Management集中蒐集與關聯日誌，偏重監控與調查查詢能力。
SOARSecurity Orchestration, Automation and Response把告警處理流程自動化（如自動開單、封鎖、通知）以降低人工作業。
SOCSecurity Operations Center（資安監控中心）負責 24/7 監控、分級、調查與應變的團隊或服務。
UEBAUser and Entity Behavior Analytics分析使用者與設備行為基線，找出異常登入與可疑操作。
IAMIdentity and Access Management（身分與存取管理）管理「誰可存取什麼」，是降低帳號濫用風險的核心。
IdP / SSO / MFA身分提供者 / 單一登入 / 多因子驗證三者常一起部署：統一身份、減少密碼疲勞、提升帳號安全。
DLPData Loss Prevention（資料外洩防護）避免敏感資料透過郵件、雲端硬碟或 USB 外流。
CASBCloud Access Security Broker（雲端存取安全代理）強化 SaaS 使用可視性與政策控管，降低 Shadow IT 風險。
MTTD / MTTRMean Time to Detect / Respond衡量「發現速度」與「處置速度」，是 SOC 成熟度關鍵指標。
RTO / RPORecovery Time Objective / Recovery Point Objective災難復原兩大指標：多久恢復、可容忍多少資料回溯。
使用建議：每次事件檢討（Post-Incident Review）都用上述術語寫成統一報告模板，可明顯提升跨部門溝通效率。

90 天入門導入節奏（建議）

第 1–2 週：盤點端點、系統版本、關鍵伺服器；決定試點群組。
第 3–6 週：PoC 比較（偵測率、誤報率、隔離流程、維運成本）。
第 7–10 週：擴大部署並建立事件分級、通報與工單流程。
第 11–12 週：做勒索/釣魚桌上演練，修正 SOP 與權責分工。

分析調查流程圖（SOC / IT 協作版）

用這個流程可以把「收到告警後該做什麼」標準化，降低遺漏與誤判。

1. 告警接收與分級
確認資產重要性、告警可信度、是否涉及關鍵帳號。
2. 初步驗證
比對時間軸、程序樹、登入來源與 IOC 命中情況。
3. 事件定級
依影響範圍與業務衝擊，分為 Low / Medium / High。
↓
4A. 高風險：立即遏止
隔離端點、停權帳號、封鎖惡意網域 / IP、保全證據。
4B. 中低風險：持續追查
擴大 Threat Hunting，確認是否有橫向移動與持久化。
↓
5. 根因分析（RCA）
釐清入口點（釣魚、漏洞、弱密碼）與控制失效原因。
6. 復原與補強
清除惡意程式、修補漏洞、重設憑證、更新偵測規則。
7. 事後檢討（PIR）
更新 SOP 與 KPI（MTTD/MTTR），輸出可重複使用的劇本。

進階落地重點（給 IT 主管與資安窗口）

  • 以 MITRE ATT&CK 建立偵測覆蓋率：至少先覆蓋 Initial Access、Credential Access、Lateral Movement。
  • 建立量化 KPI：每月追蹤 MTTD、MTTR、誤報率、隔離成功率，避免只看告警數量。
  • 分級自動化策略：低風險告警自動關聯，高風險告警自動隔離 + 人工複核。
  • 把備份演練納入資安演練：確認 RTO/RPO 能符合營運要求，才算真正有韌性。

G. 進一步學習資源（含進階路線）

公開框架與知識庫

  • NIST CSF 2.0（資安治理框架）
  • MITRE ATT&CK（攻擊技術地圖）
  • CISA 勒索軟體防護與事件回應指南

進階學習路線（90 天）

  1. 第 1–3 週：建立 ATT&CK 對照表，盤點目前可見的攻擊技術。
  2. 第 4–6 週：完成 2 個 Purple Team 劇本（勒索、帳號盜用）。
  3. 第 7–9 週：調整偵測規則，將高誤報規則優化到可營運範圍。
  4. 第 10–12 週：執行跨部門 Incident Drill，驗證通報與復原流程。
